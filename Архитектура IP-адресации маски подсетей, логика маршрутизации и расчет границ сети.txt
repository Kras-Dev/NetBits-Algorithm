    «Архитектура IP-адресации: маски подсетей, логика маршрутизации и расчет границ сети»
    ЧАСТЬ 1. Архитектура IPv4-адресации

IP-адрес — уникальный числовой идентификатор устройства в компьютерной сети. IP-адрес выполняет две основные функции:
Идентификация узла (компьютера, маршрутизатора или другого устройства) в сети.
Определение местоположения этого узла в сетевой иерархии.

    1. Базовые понятия и структура IPv4
Формат: 32 бита, разделенные на 4 октета по 8 бит каждый.
Запись: Десятичная с точками (например, 192.168.1.1) или двоичная, это 4 группы по 8 бит (всего 32 бита)(например, 11000000.10101000.00000001.00000001).
Логическая структура: Каждый адрес делится маской на адрес сети (идентификатор группы) и адрес хоста (идентификатор конкретного устройства в этой группе).
Лимит емкости: Всего существует около 4,3 миллиарда IPv4-адресов, что привело к их глобальному дефициту.

    2. Служебные адреса и ограничения
В любой IPv4-подсети два адреса зарезервированы и не могут быть выданы конечным устройствам:
Адрес сети: Самый первый адрес в диапазоне. В хостовой части все биты равны 0 (Пример: 192.168.1.0). Используется маршрутизаторами для записи в таблицах маршрутизации.
Широковещательный адрес (Broadcast): Самый последний адрес в диапазоне. В хостовой части все биты равны 1 (Пример: 192.168.1.255). Используется для отправки пакета всем узлам в данной подсети одновременно.
Формула полезной емкости: (2^N - 2), где (N) — количество хостовых бит (нулей в маске подсети).

    3. Специальные диапазоны IPv4
Localhost (Loopback): 127.0.0.0/8 (обычно 127.0.0.1). Петля обратной связи. Используется устройством для обращения к самому себе без отправки данных в физическую сеть.
Частные (серые) адреса (RFC 1918): Используются только внутри локальных сетей и не маршрутизируются в интернете:
10.0.0.0/8 (от 10.0.0.0 до 10.255.255.255)
172.16.0.0/12 (от 172.16.0.0 до 172.31.255.255)
192.168.0.0/16 (от 192.168.0.0 до 192.168.255.255)
Link-Local (APIPA): 169.254.0.0/16. Автоматически назначаются операционной системой, если устройство настроено на получение IP по DHCP, но сервер не ответил.

    4.  Логика определения сети (логика побитового «И» (AND))
Компьютер определяет границы своей сети, накладывая маску на собственный IP-адрес с помощью логического умножения. Правило: (1 * 1 = 1), любой бит при умножении на (0) дает (0).
Пример: Устройство с IP 192.168.1.35 и маской /24 рассчитывает адрес сети:
IP (192.168.1.35):   11000000.10101000.00000001.00100011
Маска (/24):         11111111.11111111.11111111.00000000
--------------------------------------------------------
Результат AND:       11000000.10101000.00000001.00000000 -> 192.168.1.0 (Сеть)

Благодаря этой операции устройство понимает: если у узла назначения адрес сети совпадает с его собственным, пакет отправляется напрямую в локальный сегмент. Если не совпадает — пакет пересылается на основной шлюз (роутер).

    5. Маски подсетей в IPv4 (CIDR)
Маска подсети — это 32-битное число, состоящее из строгой непрерывной цепочки единиц, переходящей в непрерывную цепочку нулей. Единицы определяют сетевую часть (Network ID - биты-единицы), нули — хостовую (Host ID - биты-нули)(11111111.11111111.11111111.00000000 = 255.255.255.0). 
Связь между маской и IP: Единицы в маске указывают, какие биты IP-адреса отвечают за номер сети. Нули указывают на биты конкретного устройства.
CIDR-нотация: Бесклассовая адресация. Запись маски через слэш (/), указывающий точное число бит-единиц.
/8  -> 255.0.0.0 (8 единиц, 24 нуля).
/24 -> 255.255.255.0 (24 единицы, 8 нулей).
/25 -> 255.255.255.128 (25 единиц: последний октет 10000000).
/32 -> 255.255.255.255 (все 32 бита единицы, указывает на один конкретный хост).

    ЧАСТЬ 2. Архитектура IPv6-адресации
    1. Базовые понятия и структура IPv6
Формат: 128-битный адрес, разработанный для полной замены IPv4.
Запись: Разделен на 8 групп по 16 бит (всего 128 бит) каждая (называются хекстэтами). Записывается шестнадцатеричными числами (HEX) через двоеточие.
Пример полного вида: 2001:0db8:85a3:0000:0000:8a2e:0370:7334

    2. Служебные адреса и ограничения в IPv6
Subnet-Router Anycast: Самый первый адрес подсети (где все биты хоста равны 0). В отличие от IPv4, он не запрещен. Он является валидным адресом и закрепляется за маршрутизатором этой сети.
Отсутствие классического Broadcast: В IPv6 полностью удален тип рассылки Broadcast. Его заменяет оптимизированный Multicast (многоадресная рассылка), что избавляет сеть от лишнего мусора и паразитной нагрузки.
Расчет емкости: Из-за отсутствия вычитания сетевого и широковещательного адресов емкость подсети рассчитывается по формуле (2^N).

    3. Специальные диапазоны IPv6
Loopback (Аналог Localhost): ::1/128. Заменяет огромный диапазон 127.0.0.0/8 из IPv4 одним конкретным адресом.
Link-Local адреса: fe80::/10. Автоматически генерируются каждым интерфейсом IPv6 для связи внутри одного физического сегмента сети. Без них стек IPv6 не работает.
Unique Local Addresses (ULA - Аналог серых адресов): fc00::/7 (чаще используется диапазон fd00::/8). Предназначены для маршрутизации строго внутри частных корпоративных сетей.
Global Unicast (Аналог белых адресов): 2000::/3. Публичные адреса для работы в глобальном интернете.

    4. Логика определения сети в IPv6
В IPv6 нет масок подсети в привычном виде, нет операции AND в десятичном понимании, поскольку здесь используется исключительно префиксная модель CIDR-нотация. Устройство берет длину префикса сети (например, /64) и просто отсекает указанное количество бит слева.
Все, что попало в префикс — адрес сети, все оставшиеся биты справа — уникальный идентификатор интерфейса (Interface ID). Поиск совпадений в таблице маршрутизации происходит мгновенно методом оценки наибольшего совпадения бит (Longest Prefix Match).
Возьмем IPv6-адрес в его полном (развернутом) виде:2001:0db8:85a3:008d:1234:5678:9abc:def0 /64
Разделим его ровно посередине — после первых четырех групп:
Первые 64 бита (Адрес сети) 2001:0db8:85a3:008d
Вторые 64 бита (Interface ID / Хост) 1234:5678:9abc:def0

    5. Маски подсетей в IPv6 (Префиксы)
В IPv6 полностью отсутствуют маски старого десятичного вида (вроде 255.255...). Используется только длина префикса.
Правила сокращения адресов:
Удаление ведущих нулей: В любом хекстэте нули слева отсекаются (0db8 -> db8; 0000 -> 0).
Сжатие нулевых групп (::): Последовательность идущих подряд нулевых хекстэтов заменяется на :: (Пример: 2001:db8:85a3::8a2e:370:7334). Применяется строго один раз на адрес.
Стандарт деления подсетей: Общепринятый размер абонентской подсети — /64. Первые 64 бита отданы под адрес сети, вторые 64 бита — под адрес хоста (Interface ID).

    ЧАСТЬ 3. Поиск минимальной маски между двумя IP-адресами
Минимальная маска (или Summary-маршрут / супернетинг) — это наиболее короткая длина префикса (наименьшее число единиц), которая способна объединить два конкретных IP-адреса в общую единую сеть.

    1. Пошаговый алгоритм расчета
Чтобы найти минимальную маску для двух произвольных IP-адресов, необходимо выполнить четыре шага:
Перевести оба IP-адреса в двоичную систему счисления.
Сравнить полученные биты слева направо, двигаясь от самого первого бита к последнему.
Найти точку расхождения — первый бит, в котором значения адресов перестают совпадать.
Посчитать количество совпадающих бит до точки расхождения. Это число и будет длиной минимальной маски в CIDR-нотации.

    2. Практический пример расчета для IPv4
Найдем минимальную общую маску для двух серверов с адресами 192.168.1.35 и 192.168.3.100.
    Шаг 1. Перевод в двоичный вид:
Переводим первый адрес (192.168.1.35):  11000000.10101000.00000001.00100011
Переводим второй адрес (192.168.3.100): 11000000.10101000.00000011.01100100
                    -------------------------------------------------------
                                        11111111 11111111 111111
    Шаг 2. Подсчет маски:
Итого совпало: 8 бит (1-й октет) + 8 бит (2-й октет) + 6 бит (из 3-го октета) = 22 бита.
Результат: Минимальная общая маска — /22 (в десятичном виде — 255.255.252.0). 
Суммарная сеть для этих хостов: 192.168.0.0/22.

    3. Особенности расчета для IPv6
Принцип расчета минимальной маски в IPv6 абсолютно идентичен IPv4, однако из-за шестнадцатеричной записи адреса требуют обязательного побитового разложения в области расхождения.
Пример: Даны адреса 2001:db8:a::1 и 2001:db8:b::2.
Первые два хекстэта 2001:db8 совпадают полностью (16 + 16 = 32 бита).
Третий хекстэт отличается: 000a и 000b. 
Разложим их в двоичный вид:a (десятичная 10) = 1010b (десятичная 11) = 1011
Первые 3 бита в этой четверке (101) совпадают. Расхождение наступает на последнем бите.
Расчет префикса: 32 бита (первые две группы) + 12 бит (три неизменных нуля в третьем хекстэте 000) + 3 совпавших бита = 47 бит.
Результат: Минимальный общий префикс — /47.
